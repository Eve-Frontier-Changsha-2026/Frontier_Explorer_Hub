Data Flow — End-to-End Sequences

Three core flows: Intel Submission, Heatmap Consumption, Bounty Lifecycle

▸ FLOW 1: Intel Submission (探勘者提交情報)
Explorer ──→ /submit UI ──→ intel::submit_intel() ──→ emit IntelSubmittedEvent
　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　│
　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　▼
　　　　　　　　　　　　　　　　　　　　　Event Indexer (WebSocket subscribe)
　　　　　　　　　　　　　　　　　　　　　│
　　　　　　　　　　　　　　　　　　　　　├─→ SQLite intel_reports INSERT
　　　　　　　　　　　　　　　　　　　　　└─→ Heatmap Aggregator incremental update
　　　　　　　　　　　　　　　　　　　　　　　　│
　　　　　　　　　　　　　　　　　　　　　　　　├─→ Spatial bucket recalc
　　　　　　　　　　　　　　　　　　　　　　　　├─→ K-anonymity check
　　　　　　　　　　　　　　　　　　　　　　　　└─→ heatmap_cache UPDATE
⏱ Latency: ~2-5s from submit to heatmap update (Premium) | +30min delay (Free)
▸ FLOW 2: Heatmap Consumption (用戶查看熱力圖)
User ──→ /map UI ──→ Data API GET /api/heatmap/:zoom
　　　　　　　　　　　　　│
　　　　　　　　　　　　　├─→ JWT verify (wallet signature)
　　　　　　　　　　　　　├─→ Query on-chain: subscription::is_active_premium()
　　　　　　　　　　　　　│
　　　　　　　　　　　　　▼
　　　　　　　 ┌────────────────────────┐
　　　　　　　 │ tier == Free? │
　　　　　　　 └───────┬────────┬───────┘
　　　　　　　　　 YES │　　　　│ NO
　　　　　　　　　　　 ▼　　　　▼
　　　　 Delayed data (t-30min)　Real-time data
　　　　 zoom 0-1 only　　　　　zoom 0-1-2
　　　　 count only　　　　　　　type breakdown + severity
　　　　　　　　　│　　　　　　│
　　　　　　　　　└──────┬─────┘
　　　　　　　　　　　　 ▼
　　　　　　　 deck.gl HeatmapLayer render
FREE: 模糊熱力色塊，大區域，僅數量，延遲 30min
PREMIUM: 精確熱力 + 活動類型 + 嚴重度 + 即時 + 全 zoom
▸ FLOW 3: Bounty Lifecycle (情報懸賞完整週期)
① Requester ──→ /bounties CreateBounty
　　│
　　└──→ bounty::create_bounty(region, types, reward, deadline)
　　　　　│ SUI locked in escrow
　　　　　▼
② Explorer sees bounty on board ──→ goes scouting
　　│
　　└──→ intel::submit_intel() with bounty_id reference
　　　　　│
　　　　　▼
③ Verification
　　├──→ Check: intel_type ∈ bounty.types_wanted?
　　├──→ Check: location ∈ bounty.target_region?
　　├──→ Check: timestamp ≤ bounty.deadline?
　　│
　　▼
④ Settlement ──→ bounty::verify_and_release()
　　├──→ escrow SUI → explorer
　　├──→ bounty.status = completed
　　└──→ intel flows into heatmap aggregation
⚡ Hackathon: mock verification in Explorer Hub contract | Production: delegates to Bounty_Escrow_Protocol
▸ FLOW 4: Single Intel Unlock (單次付費解鎖)
User clicks locked intel on map ──→ IntelPanel: UnlockButton
　　│
　　└──→ access::unlock_intel(intel_id, payment)
　　　　　│
　　　　　├──→ PricingTable lookup → deduct SUI
　　　　　├──→ Mint UnlockReceipt NFT → user
　　　　　├──→ Revenue split: reporter gets % (incentivize submissions)
　　　　　└──→ emit IntelUnlockedEvent
　　　　　　　　│
　　　　　　　　▼
　　　　　UI re-fetches intel detail → shows full data
💡 Revenue split creates a flywheel: more unlocks → reporters earn more → more submissions → richer heatmap → more subscribers
